Como o HTML é frequentemente a primeira experiência de pessoas com codificação, pode haver uma quantidade razoável de apreensão sobre como aprendê-lo. Felizmente, a sintaxe HTML é relativamente simples e fácil de aprender. A maioria das pessoas pode aprender os fundamentos do HTML e começar a codificá-lo no mesmo dia.

Uma linguagem de marcação

HTML é uma linguagem de marcação. Isso significa que o conteúdo da página é "marcado "por tags que identificam o conteúdo dentro delas. Um parágrafo, por exemplo, pode ser identificado colocando o "<p>" tag de abertura antes do conteúdo dos parágrafos e o "</p>" tag de fechamento no final de um parágrafo. O parágrafo completo ficaria assim: 

  <p>Este é um parágrafo </p>

As tags consistem em um colchete angular esquerdo (<) seguido por um caractere ou caracteres que identificam a tag (o "p " para parágrafo) e um colchete de ângulo reto de fechamento (>). As tags de fechamento de um elemento são exatamente iguais a uma tag de abertura, exceto que uma barra (/) irá preceder os caracteres de tags. Embora a maioria dos elementos exija uma tag de abertura e fechamento, a tag de fechamento é opcional para alguns elementos e não é obrigatória para os outros. Embora existam exceções à regra, na maior parte, qualquer elemento que contenha conteúdo dentro das tags de abertura e fechamento também requer uma tag de fechamento.

Estrutura básica do documento

O núcleo de todos os documentos HTML gira em torno de três tags básicas. Primeiro, uma tag html (<html>) é necessária para identificar o documento como um arquivo HTML. Diretamente dentro da tag html, você encontrará o elemento head (<head>). O chefe de um documento é onde você encontrará os metadados do documento, o título do documento e links para recursos externos, como folhas de estilo e scripts. Uma boa maneira de pensar sobre a cabeça do documento é que ele não contém nenhum dos conteúdos visuais da página, mas contém informações sobre o documento e os recursos que ajudam a fazer a página funcionar. Diretamente após a cabeça do documento, você encontrará seu corpo (<body>). O corpo é onde você encontrará todo o conteúdo real da página. Cabeçalhos, parágrafos, imagens, listas, tabelas e outros conteúdos serão localizados aqui. Basicamente, um arquivo HTML ficaria assim: 

 
  <html>
       <cabeça>
       </head>
       <body>
       </body>
  </html>
    

DOCTYPES

Se você já olhou para páginas HTML, provavelmente notou uma tag longa e um pouco intimidante pouco antes da tag HTML de abertura. Esta é uma declaração doctype e é um componente muito importante, mas muitas vezes incompreendido de páginas HTML. Essencialmente, ele diz ao agente usuário analisar sua página qual versão de HTML (ou XHTML) esperar, para que ele saiba quais regras de sintaxe usar ao renderizar sua página. O tipo de documento que você usa deve ser baseado na versão do HTML que você está usando para criar a página. Embora tudo soe bem na teoria, na realidade, na maioria das vezes, um tipo de documento está simplesmente acionando o "modo-padrão". em vez do "modo quirks "(com base em navegadores mais antigos não-padrão maneira de renderizar páginas). Apenas por esse motivo, todos os documentos HTML devem ser precedidos por uma declaração de tipo de documento. Para mais informações sobre doctypes, e a história por trás deles, confira a excelente seção de Mark Pilgrim sobre doctypes do seu livro Dive Into HTML5 . Aqui estão alguns dos tipos de documentos HTML mais comuns:

HTML 4.0 transitório

  <!DOCTYPE HTML PUBLIC "- // W3C // DTD HTML 4.01 Transitional // EN ""http://www.w3.org/TR/html4/loose.dtd">

XHTML 1.0 Transitional

  <!DOCTYPE html PUBLIC - // W3C // DTD XHTML 1.0 Transicional // EN ""http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

HTML5

  <!DOCTYPE HTML>

Atributos do elemento

Alguns elementos podem ser aprimorados por meio do uso de atributos. Atributos permitem que você forneça mais informações ou funcionalidades adicionais ao conteúdo. Atributos são adicionados à tag de abertura de um elemento e consistem em duas partes, o nome e o valor . Embora a sintaxe varie com base na versão do HTML que você está usando, é prática padrão colocar os valores entre aspas. 

  <h1 class = "título">Título principal do artigo</h1>

Elementos substituídos

Alguns elementos HTML representam o conteúdo que é substituído por um recurso externo, como uma imagem, um controle de formulário ou um arquivo de vídeo. Esses elementos são referidos como elementos substituídos e geralmente têm uma largura ou altura predeterminada. Em alguns casos, os elementos terão atributos que informam ao navegador onde encontrar um recurso externo, como uma imagem ou um vídeo. 

  <img src="photo.jpg" alt="minha foto incrível"> 

Estrutura de código

Documentos HTML criam estrutura ao aninhar elementos dentro um do outro. Você pode agrupar uma seção do conteúdo da página agrupando-os em um <div>ou <seção>elemento, por exemplo. Ao aninhar uma tag em outra, você deve primeiro fechar todos os elementos filhos antes de fechar qualquer elemento pai.
Essa sintaxe, por exemplo, seria incorreta : 

  <p>Você deve fechar todas as tags aninhadas <strong>primeiro!</p></strong>

Em vez disso, a sintaxe correta seria: 

  <p>Você deve fechar todas as tags aninhadas <strong>primeiro!</strong></p>

O HTML também possui regras específicas sobre quais elementos podem ser aninhados em outros elementos. Um parágrafo (<p>) por exemplo, não pode ser aninhado dentro de um título (<h1>).  Na maior parte, essas regras são baseadas no tipo de conteúdo que a tag representa. O HTML 4 tem dois tipos básicos de conteúdo: block e inline . Elementos de nível de bloco normalmente ocupam sua própria linha em um documento, geralmente empilhando um em cima do outro na ordem em que aparecem em seu código. Elementos de nível inline podem aparecer dentro do fluxo de elementos de nível de bloco. Elementos como forte (<strong>), negrito (<b>), ênfase (<em>), itálico (<i>) e span (<span>) são sempre encontrados dentro de elementos de nível de bloco .
O HTML5, por outro lado, expande o conceito de tipos de conteúdo para sete categorias diferentes de conteúdo, com alguns elementos pertencentes a várias categorias. Isso substitui o conceito de elementos de nível de bloco e inline e adiciona algumas regras adicionais de sintaxe de aninhamento. Na maioria das vezes, as regras sobre os elementos de aninhamento fazem sentido e são fáceis de entender. Você deve estar preparado, no entanto, para que leve algum tempo antes de compreender todas as regras que envolvem os elementos de aninhamento. Uma maneira de garantir que seu código seja estruturado corretamente é validar seu código por meio de um serviço como o serviço de Validação de marcação do W3C.

Comentando código

Geralmente, é útil deixar anotações para você ou para outros desenvolvedores em seu código. Talvez você queira se lembrar do que um determinado script faz, ou lembrar um colega de trabalho de como estruturar adequadamente um conteúdo específico. Para fazer isso, você usará comentários. Os comentários devem aparecer em sua própria linha e geralmente aparecem diretamente antes ou depois do código a que se referem. Os comentários começam com um colchete angular esquerdo, um ponto de exclamação e dois hífens (<! -) e terminam com dois hífens seguidos por um colchete angular (->).

Sintaxe de comentário:

  <!-- Este é um comentário -->

Usando caracteres especiais

Certos caracteres são reservados em HTML, o que significa que você não deve usá-los fora de sua finalidade específica. Os colchetes angulares, por exemplo, são reservados para tags, e usá-los em qualquer lugar fora de uma tag pode causar erros de análise em navegadores. No entanto, você ainda pode usar esses caracteres reservados e outros caracteres especiais ou símbolos usando o que é conhecido como entidade de caractere nomeado . Esses códigos especiais dizem ao agente do usuário para substituir o código da entidade por um caractere específico. Essas entidades começam com um "e" comercial (&) seguido pelo nome da entidade e terminam com um ponto e vírgula (;). Para exibir um e comercial, por exemplo, você digitaria &amp; no lugar do "e" comercial real. Você pode encontrar listas de entidade de caractere nomeadas dentro das especificações HTML, embora o formato para elas possa ser um pouco difícil de ler. Você também pode encontrar uma lista útil de entidades de caracteres na página Referência HTML e uma lista mais abrangente na Wikipedia.
